# Lovable-Specific Fit

## Shared primitive
Lovable Skills = portable, readable, editable **markdown playbooks**. Einstein Brain nodes = portable **markdown** with provenance frontmatter. **Same shape.** Brain is not a competing format — it's the trust layer around the format.

## What Brain adds on top of Skills/Knowledge
| Lovable has | Einstein Brain adds |
|---|---|
| Skills (markdown playbooks) | **Provenance** — is this verified vs machine-generated, when last checked |
| Knowledge (connected sources) | **Abstention** — agent says "I don't know" instead of guessing |
| Agents that read + act | **Human-gated writeback** — a new skill/fact is quarantined until a person stamps it |
| Connectors (Google Workspace, Gemini Enterprise) | **The trust index** that decides which connected knowledge becomes *verified, reusable* memory/skills |

## The exact framing line (approved)
> "Lovable already connects to Google Workspace and Gemini Enterprise; Einstein Brain is the trust layer that decides what company knowledge becomes verified, reusable memory/skills."

## How they compose
```
Lovable connectors  →  raw company knowledge
        →  Einstein Brain: verify + provenance + abstention + human stamp
        →  trusted nodes  →  compiled into Lovable Skills (executable)
        →  Lovable agents act on TRUSTED skills, and abstain when unsure
```
A Lovable Skill becomes the **executable output** of a Brain-verified node — closing the YC "knowledge → executable skill" loop, with trust built in.

## Why Lovable specifically
- Ships agents/apps at huge scale → hallucination + untrusted knowledge is their customers' #1 enterprise blocker; a trust layer is a wedge into enterprise-grade deployments.
- Fast-scaling team → their own internal-ops memory is a live use case.
- "Verified, abstaining memory" is a feature their users will ask for on generated apps.

## The partnership, not the overlap
Brain makes Lovable Skills **trustworthy at enterprise scale**. Complementary surface — we don't replace Knowledge or Skills, we make them safe to rely on.
