# Research Sources

Primary/official sources behind the framing in this repo. Accessed 2026-06-18. Quotes are verbatim where marked; undatable pages are access-dated only.

## YC — the demand signal
- **YC Requests for Startups** — https://www.ycombinator.com/rfs (current Summer-2026 RFS; accessed 2026-06-18).
  - Entry **"Company Brain"** (Tom Blomfield), verbatim: *"The biggest blocker to AI automation of companies is no longer the models, they just got so good so quickly. Now the blocker is the domain knowledge."*
  - Entry **"The AI Operating System for Companies"** (Diana Hu), verbatim: *"The best AI-native companies we're seeing have figured out something most haven't: they've made their entire company queryable."*

## GitHub — repo presentation
- **About READMEs** — https://docs.github.com/en/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/about-readmes (accessed 2026-06-18). A README typically covers: what the project does, why it's useful, how to get started, where to get help, who maintains it.
- **Best practices for repositories** — https://docs.github.com/en/repositories/creating-and-managing-repositories/best-practices-for-repositories (accessed 2026-06-18): a README plus license, contribution guidelines, and a code of conduct "communicates expectations … and helps you manage contributions." A `SECURITY.md` guides responsible vulnerability disclosure.

## Lovable — Skills + GitHub import
- **Define reusable instructions with skills** — https://docs.lovable.dev/features/skills (accessed 2026-06-18).
  - A skill is a folder with a main `SKILL.md`; requires a name (lowercase/numbers/hyphens, 1–64 chars), a description (the trigger), and markdown instructions.
  - Verbatim: *"Knowledge is always included in context… Skills are loaded on demand when the request matches."*
  - GitHub import (verbatim): *"Choose Add → Import from GitHub, and paste a public GitHub repository URL."* (This is Skills-level import; broader project↔GitHub codebase sync is a separate Lovable feature, not covered here.)

## Notes on rigor
- Claims about *what incumbents lack* (abstention, human-gated knowledge writeback) are summarized from a separate dated competitor scan and stated cautiously ("I haven't found an incumbent shipping…"). This repo deliberately avoids broad competitor claims.
- Metrics in this repo are from a private eval set, re-runnable on the private system only.
