# YC Alignment

YC's current (Summer-2026) Requests for Startups names this exact space. Honest mapping of what's met vs roadmap.

## "Company Brain" (Tom Blomfield)
> "pull knowledge out of all fragmented sources, structure it, keep it current, and turn it into an **executable skills file** for AI … the missing layer between raw company data and reliable AI automation."

| YC ask | Einstein Brain | Status |
|---|---|---|
| Structure company know-how | typed markdown nodes + provenance + wikilink graph | ✅ real |
| Keep it current | `last_verified` / staleness flags | 🟡 partial (labels over-broad, being calibrated) |
| Pull from fragmented sources | manual/assisted ingest | 🟡 partial (connectors = roadmap) |
| Turn into **executable skills file** | retrieval today; skill-generation roadmap | 🔴 roadmap |
| "missing layer … reliable automation" | abstention + provenance + human gate | ✅ the trust half is real |
| Not search / not a doc-chatbot | abstains, provenance-weighted, governed | ✅ |

## "AI Operating System for Companies" (Diana Hu)
> "a single intelligence layer that can reason across [Slack/Linear/GitHub/Notion/calls] … generate specs agents can execute on."

| YC ask | Status |
|---|---|
| Make company queryable | ✅ for curated knowledge; live ops systems = roadmap |
| Connect tickets/GitHub/meetings | 🔴 roadmap (manual today) |
| Closed loop (monitor→compare→adjust) | 🔴 roadmap |
| Generate specs agents execute | 🔴 roadmap |

## Honest verdict
Einstein Brain is **YC-adjacent**: it owns the *trust* half YC names — verified, abstaining, human-gated memory — which **I haven't found an incumbent shipping**. The *executable-skill + connector + closed-loop* half is roadmap. We pitch the real half and name the roadmap; we don't claim "complete company brain."
